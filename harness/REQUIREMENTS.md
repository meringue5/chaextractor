# chaextractor 요구사항 하네스

작성일: 2026-05-17

## 역할

이 문서는 `AGENTS.md`에 섞여 있던 제품 요구사항을 하네스 관점에서 재정리한 정본이다. 기능을 추가하거나 공개 약속을 바꿀 때는 이 문서를 먼저 갱신하고, 사용자 문구가 바뀌면 `README.md`, 작업 지침이 바뀌면 `AGENTS.md`도 함께 맞춘다.

## 기능 요구사항

| 영역 | 요구사항 | 현재 상태 |
|---|---|---|
| 입력 | ZIP/TXT/CSV 또는 폴더 입력을 받아 대화 로그와 첨부파일을 파싱한다. | 요구사항 |
| 플랫폼 | iOS/Android/Windows/macOS 카카오톡 내보내기 파일을 지원한다. | 요구사항 |
| 날짜 | 파싱 결과를 날짜 단위로 그룹화하고 날짜별 탐색을 제공한다. | 요구사항 |
| 시스템 메시지 | 입장/퇴장, 날짜 구분선, 삭제 메시지 등 시스템 메시지는 표시 대상에서 제외한다. | 요구사항 |
| 병합 | 동일 사용자의 연속 텍스트 발화는 하나의 발화로 병합한다. | 요구사항 |
| 메시지 타입 | 텍스트, 사진, 파일, 이모티콘을 구분한다. | 요구사항 |
| 첨부파일 | 플랫폼 규칙에 따라 메시지와 첨부파일을 매핑하고, 누락되어도 앱이 중단되지 않는다. 사진 파일이 누락된 경우 원본 대화에서 사진을 열어 내려받은 뒤 다시 내보내라는 안내를 제공한다. | 요구사항 |
| 검색 | 전체 메시지를 대상으로 검색하고 날짜 목록과 결과에 반영한다. | 요구사항, 하이라이트 구현은 드리프트 |
| 통계 | 날짜별 메시지 수, 참여자 수, 필터 대상 사용자 발언 수, 사진 수를 제공한다. | 요구사항 |
| 사용자 필터 | 기본값 `채상욱 리더`를 가진 필터 대상 사용자 입력, 대상 발언 강조, 스크롤 마커, 대상 사용자 필터를 제공한다. | 요구사항 |
| 갈무리 | 선택 날짜 또는 전체 대화를 LLM 요약에 붙여넣기 좋은 TXT로 정리하고, 복사와 다운로드를 제공한다. 첨부파일 내용과 이미지 base64는 포함하지 않는다. | 요구사항 |
| 설정 | 기본 테마는 1995이며, Light/Dark/1995/System 테마와 폰트 설정은 즉시 반영되고 선택 상태가 유지된다. | 요구사항 |
| 보조 콘텐츠 | 메인 화면 오른쪽 링크 사이드바에서 머니버스 꿀팁과 버그 제보 진입점을 제공한다. | 요구사항 |
| 오류 보고 | JS 오류와 파일 처리 실패를 감지하면 오류 보고 모달을 즉시 열고, 오류 메시지, 처리 단계, 입력 파일/ZIP 구조, 대화 파일 검증 결과와 샘플 라인이 들어간 진단 리포트를 만든다. 전체 리포트는 복사 또는 TXT 다운로드가 가능하며, Google Form 내용 칸에는 URL 길이 제한을 피한 오류 요약을 자동 입력한다. | 요구사항 |

## 화면 요구사항

### `setupScreen`

초기 화면은 첫 실행 또는 파일 처리 전 상태에서 표시된다.

필수 구성:

- 사용 가이드 영역
- ZIP/TXT/CSV 파일 선택 버튼
- 폴더 선택 버튼
- 드래그앤드롭 영역
- 진행률 표시
- 처리 완료 후 시작 버튼
- 진단 리포트가 사전 입력된 Google Form 버그 제보 버튼
- 오류 보고 모달

필수 흐름:

1. 사용자가 ZIP/TXT/CSV 파일, 폴더, 드래그앤드롭 중 하나로 파일을 입력한다.
2. 처리 진행률과 상태 메시지를 표시한다.
3. 처리 완료 후 시작 버튼을 활성화한다.
4. 시작 버튼 클릭 시 메인 화면으로 전환한다.

### `app`

메인 화면은 파일 처리 완료 후 시작 버튼 클릭으로 진입한다.

필수 구성:

- 좌측 탐색 패널
- 대화 뷰포트
- 우측 링크 패널
- 이미지 확대 모달
- 설정 모달
- 모바일 좌측/우측 사이드바 토글과 오버레이

좌측 탐색 패널 필수 기능:

- 통계 요약
- 검색 입력
- 월별 캘린더
- 날짜 목록
- 사용자 필터 토글과 필터 대상 입력
- 설정 모달 열기

우측 링크 패널 필수 기능:

- 머니버스 꿀팁과 유용한 팁 외부 링크 목록
- 진단 리포트가 사전 입력된 Google Form 버그 제보 버튼
- 외부 링크는 사용자가 명시적으로 클릭했을 때만 열린다.

대화 뷰포트 필수 기능:

- 선택된 날짜의 메시지 렌더링
- 날짜별 통계 헤더
- 메시지 사용자명/시간 표시
- 첨부파일 지연 로딩
- 이미지 클릭 확대
- 누락된 사진 자리에서 데스크톱 hover와 모바일 tap으로 복구 안내 제공
- 필터 대상 사용자 발언 강조
- 필터 대상 사용자 발언 위치 스크롤 마커
- 갈무리 버튼으로 현재 날짜 또는 전체 대화 TXT 복사/다운로드

모바일 요구사항:

- 900px 이하에서 사이드바는 슬라이드 패널로 동작한다.
- 사이드바가 열렸을 때 오버레이로 대화 영역과 구분한다.
- 모바일에서는 대화 뷰포트, 좌측 탐색 패널, 우측 링크 패널 중 하나만 화면을 점유한다.
- 모바일에서 좌측 탐색 패널을 열면 우측 링크 패널은 닫히고, 우측 링크 패널을 열면 좌측 탐색 패널은 닫힌다.
- 검색/날짜 탐색과 대화 읽기 흐름이 터치 환경에서 유지되어야 한다.

## 비기능 요구사항

| 영역 | 요구사항 | 하네스 필요 여부 |
|---|---|---|
| 브라우저 호환 | 최신 Chrome, Safari(iOS), Edge, Firefox에서 동작한다. | Browser smoke |
| 기능 제한 안내 | `File`, `Blob`, `IndexedDB`, `URL.createObjectURL` 미지원 환경에서는 복구 가능한 안내를 제공한다. | Capability check |
| 성능 | 데스크톱 기준 50만 메시지 규모까지 10초 내 파싱 완료를 목표로 한다. | Performance smoke |
| 렌더링 | 날짜 단위 렌더링과 첨부파일 지연 로딩으로 메모리 피크를 줄인다. | Browser/perf smoke |
| 접근성 | 검색, 날짜 선택, 모달 닫기 등 주요 기능은 키보드로 사용할 수 있어야 한다. | Accessibility smoke |
| 접근성 | 포커스 표시와 4.5:1 이상 대비를 유지한다. | Visual/a11y check |
| 안정성 | 잘못된 파일, 인코딩 문제, 누락 첨부파일은 복구 가능한 오류로 처리한다. | Error fixture |
| 안정성 | JS 오류와 처리 실패는 오류 보고 모달을 즉시 열고 복사 가능한 리포트로 이어진다. | Diagnostic report smoke |
| 개인정보 | 대화 원문, 첨부파일, 파싱 결과는 자동으로 외부 서버에 전송하지 않는다. | Network surface check |
| 진단 품질 | 진단 리포트는 파싱 실패 원인 확인에 필요한 파일명, 경로, 크기, 후보 파일 검증 결과, 샘플 라인을 포함한다. | Diagnostic report smoke |
| 개인정보 | 갈무리 TXT는 사용자가 명시적으로 복사/다운로드할 때만 생성되며, 첨부파일 내용과 이미지 base64를 포함하지 않는다. | UI smoke |

개인정보와 보안의 세부 표준은 [MANIFEST.md](MANIFEST.md)를 우선한다.

## 요구사항 변경 규칙

1. 새 기능이 사용자에게 보이는 약속이면 이 문서에 먼저 추가한다.
2. 플랫폼 파서 요구사항은 [DOMAIN_RULES.md](DOMAIN_RULES.md)와 fixture/expected 결과를 함께 갱신한다.
3. 구현과 요구사항이 다르면 [MANIFEST.md](MANIFEST.md)의 구현-only 또는 미결정 항목으로 먼저 분류한다.
4. 공개 문구가 바뀌면 `README.md`를 갱신한다.
5. 작업 완료 후 `HISTORY.md`에는 변경 이력과 검증 결과만 남긴다.
