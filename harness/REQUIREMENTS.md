# chaextractor 요구사항 하네스

작성일: 2026-05-17

## 역할

이 문서는 `AGENTS.md`에 섞여 있던 제품 요구사항을 하네스 관점에서 재정리한 정본이다. 기능을 추가하거나 공개 약속을 바꿀 때는 이 문서를 먼저 갱신하고, 사용자 문구가 바뀌면 `README.md`, 작업 지침이 바뀌면 `AGENTS.md`도 함께 맞춘다.

## 화면 요구사항

### `setupScreen`

초기 화면은 첫 실행 또는 파일 처리 전 상태에서 표시된다.

필수 구성:

- 사용 가이드 영역
- ZIP 선택 버튼
- 폴더 선택 버튼
- 드래그앤드롭 영역
- 진행률 표시
- 처리 완료 후 시작 버튼
- 처리 완료 후 히어로 이미지
- 문의/제보 외부 링크

필수 흐름:

1. 사용자가 ZIP, 폴더, 드래그앤드롭 중 하나로 파일을 입력한다.
2. 처리 진행률과 상태 메시지를 표시한다.
3. 처리 완료 후 시작 버튼을 활성화한다.
4. 시작 버튼 클릭 시 메인 화면으로 전환한다.

### `app`

메인 화면은 파일 처리 완료 후 시작 버튼 클릭으로 진입한다.

필수 구성:

- 좌측 탐색 패널
- 대화 뷰포트
- 우측 링크 패널
- 이미지 확대 모달
- 설정 모달
- 모바일 좌측/우측 사이드바 토글과 오버레이

좌측 탐색 패널 필수 기능:

- 통계 요약
- 검색 입력
- 월별 캘린더
- 날짜 목록
- 리더 필터 토글
- 설정 모달 열기

우측 링크 패널 필수 기능:

- 머니버스 꿀팁 외부 링크 목록
- 문의/제보 외부 링크
- 외부 링크는 사용자가 명시적으로 클릭했을 때만 열린다.

대화 뷰포트 필수 기능:

- 선택된 날짜의 메시지 렌더링
- 날짜별 통계 헤더
- 메시지 사용자명/시간 표시
- 첨부파일 지연 로딩
- 이미지 클릭 확대
- 리더 발언 강조
- 리더 발언 위치 스크롤 마커

모바일 요구사항:

- 900px 이하에서 사이드바는 슬라이드 패널로 동작한다.
- 사이드바가 열렸을 때 오버레이로 대화 영역과 구분한다.
- 모바일에서는 대화 뷰포트, 좌측 탐색 패널, 우측 링크 패널 중 하나만 화면을 점유한다.
- 모바일에서 좌측 탐색 패널을 열면 우측 링크 패널은 닫히고, 우측 링크 패널을 열면 좌측 탐색 패널은 닫힌다.
- 검색/날짜 탐색과 대화 읽기 흐름이 터치 환경에서 유지되어야 한다.

## 비기능 요구사항

| 영역 | 요구사항 | 하네스 필요 여부 |
|---|---|---|
| 브라우저 호환 | 최신 Chrome, Safari(iOS), Edge, Firefox에서 동작한다. | Browser smoke |
| 기능 제한 안내 | `File`, `Blob`, `IndexedDB`, `URL.createObjectURL` 미지원 환경에서는 복구 가능한 안내를 제공한다. | Capability check |
| 성능 | 데스크톱 기준 50만 메시지 규모까지 10초 내 파싱 완료를 목표로 한다. | Performance smoke |
| 렌더링 | 날짜 단위 렌더링과 첨부파일 지연 로딩으로 메모리 피크를 줄인다. | Browser/perf smoke |
| 접근성 | 검색, 날짜 선택, 모달 닫기 등 주요 기능은 키보드로 사용할 수 있어야 한다. | Accessibility smoke |
| 접근성 | 포커스 표시와 4.5:1 이상 대비를 유지한다. | Visual/a11y check |
| 안정성 | 잘못된 파일, 인코딩 문제, 누락 첨부파일은 복구 가능한 오류로 처리한다. | Error fixture |
| 개인정보 | 대화 원문, 첨부파일, 파싱 결과는 자동으로 외부 서버에 전송하지 않는다. | Network surface check |

개인정보와 보안의 세부 표준은 [MANIFEST.md](MANIFEST.md)를 우선한다.

## 요구사항 변경 규칙

1. 새 기능이 사용자에게 보이는 약속이면 이 문서에 먼저 추가한다.
2. 플랫폼 파서 요구사항은 [DOMAIN_RULES.md](DOMAIN_RULES.md)와 fixture/expected 결과를 함께 갱신한다.
3. 구현과 요구사항이 다르면 [MANIFEST.md](MANIFEST.md)의 구현-only 또는 미결정 항목으로 먼저 분류한다.
4. 공개 문구가 바뀌면 `README.md`를 갱신한다.
5. 작업 완료 후 `HISTORY.md`에는 변경 이력과 검증 결과만 남긴다.
