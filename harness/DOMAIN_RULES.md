# chaextractor 도메인 규칙 하네스

작성일: 2026-05-17

## 역할

이 문서는 카카오톡 내보내기 파일의 플랫폼별 규칙을 정리한 정본이다. 기존 에이전트 작업 지침에 있던 도메인 지식과 `HISTORY.md`에 누적된 발견사항 중, 앞으로도 구현 기준으로 써야 하는 내용을 이곳으로 옮겼다.

## 플랫폼 지원 상태

| 플랫폼 | 상태 | 판정 |
|---|---|---|
| iOS | 공식 지원 | 요구사항 |
| Android | 공식 지원 | 요구사항 |
| Windows | 공식 지원 | 요구사항 |
| macOS | 공식 지원 | 요구사항 |

Windows는 데스크톱 내보내기 텍스트 로그 파싱을 공식 지원한다. Windows 첨부파일 매핑은 실제 샘플과 규칙이 확인되기 전까지 공식 범위에 포함하지 않는다. 현재 `windows-attachments-unsupported` fixture는 첨부파일이 매핑되지 않는 상태를 회귀로 고정한다.
macOS는 데스크톱 CSV 텍스트 로그 파싱을 공식 지원한다. macOS 첨부파일 매핑은 실제 샘플과 규칙이 확인되기 전까지 공식 범위에 포함하지 않는다.

## 대화 파일명 규칙

| 플랫폼 | 파일명/형식 | 상태 |
|---|---|---|
| iOS | `Talk_YYYY.M.D HH_mm-n.txt` | 요구사항 |
| Android | `KakaoTalkChats.txt` | 요구사항 |
| Windows | `KakaoTalk_YYYYMMDD_HHMM_SS_n_*.txt` | 요구사항 |
| macOS | `KakaoTalk_Chat_[방 이름]_YYYY-MM-DD-HH-MM-SS.csv` 또는 `Date,User,Message` 헤더 CSV | 요구사항 |

iOS 월/일은 한 자리일 때 0 패딩이 없고, 날짜와 시간 사이에 공백이 있다.

## 첨부파일 규칙

| 플랫폼 | 패턴 | 매핑 규칙 | 상태 |
|---|---|---|---|
| iOS | `YYYYMMDD_HHMMSS(_n)?.(jpeg|jpg|png|webp|pdf)` | 메시지 시간 기준 가까운 첨부파일 매핑, 기본 허용 범위 ±30분 | 요구사항 |
| Android | `{64자리 hex}.(jpg|jpeg|png|gif|webp)` | 대화 내용의 파일명을 `attachment_ref`로 직접 매핑 | 요구사항 |
| Android 일반 파일/PDF | `파일: {파일명}`. 파일명은 URL 인코딩될 수 있음 | URL 인코딩/디코딩 비교 후 직접 매핑 | 요구사항 |
| Windows | 미확정 | 실제 export 구조 확인 전까지 미지원 | 미결정/TODO |
| macOS | 미확정 | 실제 export 구조 확인 전까지 미지원 | 미결정/TODO |

공통 규칙:

- 대화 내에 첨부파일명이 있어도 실제 ZIP/폴더에 없을 수 있다.
- 누락 첨부파일은 오류로 앱을 중단하지 않고 메시지 단위의 누락 상태로 표시한다.
- 대화 내용, 사용자명, 파일명, 첨부파일 참조 문자열은 모두 신뢰하지 않는 입력이다.

## 시스템 메시지 규칙

표시 대상에서 제외해야 하는 항목:

- 카카오톡 내보내기 제목/저장일/빈 줄 헤더
- 날짜 구분선
- 입장/퇴장 메시지
- `메시지가 삭제되었습니다.`
- macOS CSV에서 `Date`/`User`가 비어 있는 `메시지가 삭제되었습니다.`, `관리자가 메시지를 가렸습니다.`
- Android의 사용자 없는 시간 줄: `YYYY년 M월 D일 오전/오후 H:mm`

플랫폼별 입장/퇴장 후보:

| 플랫폼 | 예시 |
|---|---|
| iOS | `YYYY. M. D. HH:mm: 사용자님이 들어왔습니다.` |
| Android | `YYYY년 M월 D일 오전/오후 H:mm, 사용자님이 들어왔습니다.` |
| Windows | `사용자님이 들어왔습니다.`, `사용자님이 나갔습니다.` |
| macOS | CSV에서 `Date`/`User`가 빈 시스템 행 |

## 일반 메시지 규칙

### iOS

기본 패턴:

```text
YYYY. M. D. HH:mm, 사용자 : 발언 내용
```

지원해야 하는 내용:

- 텍스트 메시지
- `사진`
- `파일: {원본파일명}`
- `이모티콘`
- 24시간 표기
- 오전/오후 표기 후보

### Android

기본 패턴:

```text
YYYY년 M월 D일 오전/오후 H:mm, 사용자 : 발언 내용
```

지원해야 하는 내용:

- 텍스트 메시지
- `{64자리 hex}.jpg` 같은 이미지 파일명
- `파일: {파일명}`. 파일명은 URL 인코딩될 수 있음
- `이모티콘`
- 연속 사진 줄. 첫 줄 이후에는 사용자/시간 없이 파일명만 나열될 수 있음

연속 사진 예시:

```text
2026년 2월 8일 오후 3:18, 테스터 : f5c8fbdb...jpg
5bb9f52bba8...jpg
f92f6c1f66...jpg
```

### Windows

기본 패턴:

```text
[사용자] [오전/오후 H:MM] 내용
```

날짜 구분선:

```text
--------------- YYYY년 M월 D일 d요일 ---------------
```

지원해야 하는 내용:

- 텍스트 메시지
- `사진 N장`
- `파일: {원본파일명}`. 단, Windows 첨부파일 실제 매핑은 아직 공식 범위 밖이다.
- `이모티콘`
- 오전/오후 표기

### macOS

기본 형식:

```csv
Date,User,Message
YYYY-MM-DD HH:mm:ss,사용자,발언 내용
```

지원해야 하는 내용:

- 텍스트 메시지
- `사진`
- `사진 N장`
- `파일: {원본파일명}`. 단, macOS 첨부파일 실제 매핑은 아직 공식 범위 밖이다.
- `이모티콘`
- CSV quoted field 안의 쉼표와 줄바꿈
- `Date`/`User`가 비어 있는 삭제/관리자 숨김 시스템 행 제외

## 파싱 불변식

파서 변경 시 반드시 지켜야 하는 기준:

- 날짜는 `YYYY-MM-DD`로 정규화한다.
- 시간은 `HH:mm` 24시간 형식으로 정규화한다.
- 메시지는 날짜별로 그룹화한다.
- 날짜 목록은 최신 날짜부터 표시한다.
- 시스템 메시지는 결과 메시지에 포함하지 않는다.
- 동일 사용자 연속 텍스트만 병합한다.
- 사진/파일/이모티콘은 텍스트와 별도 메시지 타입으로 유지한다.
- 필터 대상 사용자 발언은 사용자명 exact match로 판정하며, 기본 대상은 `채상욱 리더`다.
- URL 포함 여부는 메시지별로 안정적으로 계산한다.
- 짧은 실제 대화 파일을 지원할지 여부는 별도 요구사항으로 결정해야 한다.

## Fixture 규칙

플랫폼 규칙을 공식 요구사항으로 승격하려면 최소한 아래 파일이 필요하다.

| 플랫폼 | 필요한 fixture |
|---|---|
| iOS | 날짜 헤더, 텍스트, 사진, PDF, 누락 첨부파일, 연속 텍스트 병합 |
| Android | `KakaoTalkChats.txt`, 64자리 hash 이미지, 연속 사진, URL 인코딩 일반 파일명 |
| Windows | 데스크톱 txt, 날짜 헤더, 오전/오후 메시지, 시스템 메시지, 연속 텍스트 병합 |
| macOS | 실제 확인 형식 기반 CSV 헤더, 24시간 timestamp, 삭제/관리자 숨김 시스템 행, CSV quoted multiline, 연속 텍스트 병합 |

각 fixture에는 메시지 수, 날짜 수, 참여자 수, 타입별 수, 첨부파일 매핑 수를 담은 expected JSON이 따라야 한다.
